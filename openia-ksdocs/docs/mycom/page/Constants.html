Constants, Acronyms, and Conventions

Credential status convention:

  • -1: Authentication process failed
  •  0: Initialized authentication process
  •  1: User identified by the IDP
  •  2: Success login
  •  3: Success logout
  •  4: Blocked user

SP Verify Options:

  • 0: The credential ID and Secret will not be verified, so if a new SP make a request to the AS without a credential value, a new credential record will be created based on the device ID or request header user agent field.
  • 1: The credential ID and Secret will be verified, but in a weak mode where it is allowed auto signed certificates.
  • 2: The credential ID and Secret will be verified, usually it should be send in the request header authorization field.

Log level convention:

  • -1: NONE  - Show none or no logs
  •  0: ALL       - Show all kind of logs
  •  1: ERROR - Show only error
  •  2: WARN  - Show errors and warnings
  •  3: INFO    - Show errors, warnings, and informational logs
  •  4: DEBUG - Show errors, warnings, informational and debug (logs in develop mode)

These values called 'logs level' define the types of messages or event records that will be displayed in the log system. These are structured in a hierarchy, which allows this flow of information to be managed in a comfortable way. You can see below the configuration of log levels associated with environments:

  • Local environment: 0 or All
  • DEV environment: 4 or Debug
  • STG environment: 4 or Debug
  • PRD environment: 1 or Error
  • TST environment: -1 or None

Note that this setting affects the entire Auth server. This is something that can be managed from the configuration files.

AS User Action:

  • 1: Only read actions will be allowed.
  • 2: Allows read and update actions.
  • 3: Allows reading and creating actions.
  • 4: Allows read and write actions (create, update)

Auth Protocols:

  • BASIC
  • SAML
  • OAUTH_PKCE
  • OAUTH_AUTHORIZATION_CODE
  • OAUTH_CLIENT_CREDENTIAL
  • OAUTH_ROPC
