Monitoring: Alerts

The alerting system for the authentication API integrates with Microsoft Teams and New Relic to provide real-time monitoring and notifications. It is important to mention that this section is complemented by Logs and FAQs

When critical events occur within the authentication process, the system triggers alerts through two channels:

  1. Microsoft Teams Channel:

    • The system sends messages to a designated Microsoft Teams channel. These messages provide instant visibility to the authentication team.
    • Alerts are structured with relevant information, such as the nature of the event and its severity.
    • Team members can collaborate on troubleshooting and resolution directly within the Teams channel.

    Note that some of these alerts define several properties that describe them. In these examples you can see the environment where the error was detected and its classification, as well as the associated flow identifier so that it can be tracked from the log records, the domain name, the company, and group identifier, among others.

    The MS Teams group where the alerts are exposed can be specified through an environment variable knows as CHANNEL_URL_OUT.

  2. New Relic:

    • The system logs events and anomalies related to the authentication API in New Relic.
    • New Relic's logging platform collects, analyzes, and visualizes these logs, enabling thorough monitoring and analysis.
    • Alerts can be configured within New Relic to automatically trigger notifications based on predefined conditions, ensuring rapid response to issues.

In summary, the alerting system creates a robust safety net for the authentication API by utilizing Microsoft Teams for immediate team awareness and collaboration, and New Relic for in-depth monitoring and automated alerting. This proactive approach enhances the system's reliability and helps ensure timely response to potential authentication issues.
